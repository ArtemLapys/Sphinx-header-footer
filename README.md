# Кастомная автогенерируемая Sphinx документация с шапкой и футером сайта 

Документация проекта, построенная на движке **Sphinx** с использованием темы **ReadTheDocs**.
В проект интегрированы кастомная шапка и футер, визуально повторяющие основной сайт.

## 1. Старт

1.1. Установите Python 3.x
1.2. Создать виртуальное окружение
```bash
python3 -m venv venv
```

1.3. Активировать виртуальное окружение
```bash
source venv/bin/activate # (MacOS/Linux)
venv\Scripts\activate # (Windows)
```

1.4. Установить зависимости
```bash
pip install -r requirements.txt
```

## 2. Сборка документации
Генерация HTML-файлов:

`make html`

Готовый сайт будет лежать в папке: `build/html/index.html`.
Для очистки старой сборки используйте `make clean`.

## 3. Особенности проекта
Проект использует JS-инъекцию для вставки шапки и футера, чтобы не переопределять сложные Jinja2-шаблоны темы.
Все кастомные файлы лежат в `source/_static/`:
-`custom.js` — Содержит HTML-код шапки/футера и логику их вставки.
-`custom.css` — Содержит стили для шапки/футера и фиксы лейаута Sphinx.

## 4. Важный моменты при разработке

### Где менять HTML:
Откройте `source/_static/custom.js`. Там есть функции `getHeaderHTML()` и `getFooterHTML()`. Меняйте верстку внутри этих строковых переменных.


### Где менять CSS:
Откройте `source/_static/custom.css`. Вы можете менять стили шапки и футера.


### ⛔ ЧТО НЕЛЬЗЯ УДАЛЯТЬ:
В файле `custom.css` есть блок `/* --- SPHINX FIX --- */`.
Не удаляйте стили для классов `.wy-grid-for-nav, .wy-nav-side` и `.wy-nav-content-wrap`.
Они отвечают за то, чтобы стандартное меню документации корректно вставало под новую шапку и не перекрывало её.


### Мобильное меню:
В `custom.js` реализована базовая логика открытия мобильного меню (класс `.burger-btn`). Если будете менять классы в HTML, не забудьте обновить селекторы в JS (функция `initInteractivity`).

## 5. Структура проекта

```bash
.
├── Makefile             # Команды сборки
├── requirements.txt     # Зависимости
├── source/
│   ├── conf.py          # Конфигурация Sphinx (подключение JS/CSS)
│   ├── index.rst        # Главная страница документации
│   ├── _static/
│   │   ├── custom.css   # <--- Стили шапки и фиксы Sphinx
│   │   └── custom.js    # <--- HTML шапки и логика вставки
│   └── ...
└── build/               # Сюда генерируется готовый сайт (в git не попадает)
```
